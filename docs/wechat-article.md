# 我做了一个 AI 自动生成 PPT 的工作流，开源了

市面上的 AI PPT 工具越来越多，但用下来总觉得差点意思：模板千篇一律、生成的内容和自己的实际材料对不上、导出的文件还不能编辑。最让我头疼的是——它们都是封闭系统，没法接入我自己的知识库。

所以我干脆自己搭了一套。跑通之后觉得挺实用，就整理成了开源项目。

---

## 它能做什么

一句话总结：你告诉 AI "做一个关于 XX 的 PPT"，它会自动走完从选题到生成 .pptx 文件的全流程，中间不需要你手动操作任何工具。

完整的流程是这样的：

```
Markdown 大纲 → 需求确认 → 风格选择（15种）→ HTML 幻灯片 → AI 配图 → PPTX → 质量审核
```

整个过程由 AI Agent 编排，你只需要在开头说清楚主题、受众、风格偏好，剩下的交给它。

---

## 和其他 AI PPT 工具有什么不同

**1. 可以接入你的本地知识库**

这是我觉得最有价值的地方。其他 AI PPT 工具只能用它们自己的知识，生成的内容往往空洞、和你的实际情况对不上。

这套工作流运行在 Claude Code（或类似的 AI 编程代理）里，意味着 AI 可以读你的本地文件、飞书文档、笔记、内部资料——然后基于这些真实内容来写 PPT。做一个给自家团队看的汇报，内容深度完全不一样。

**2. 15 种精选高质量风格**

不是那种泛用的通用模板，而是参考真实产品调性手工定制的风格，包括：

- Apple Bento Grid（产品发布、科技展示）
- TED Talk Minimal（演讲、思想传播）
- Netflix Dark Entertainment（创意展示、品牌故事）
- McKinsey Consulting Blue（商业汇报、咨询报告）
- Silicon Valley Pitch（创业路演）
- Wabi-Sabi（禅意东方美学）
- 发布会全图大字（TED 式演讲）

……等共 15 种。每种风格都有明确的适用场景。

**3. 输出真正可编辑的 .pptx 文件**

生成的是标准 PowerPoint 文件，可以直接在 PowerPoint 或 Keynote 里打开修改。不是截图，不是 PDF，不是只能在某个平台里用的专有格式。

**4. 三种配图方式**

- 本地文件（直接用你自己的图）
- AI 生成（调用 Gemini）
- 免费图库（Pexels）

配图逻辑是基于情绪匹配，而不是关键词堆砌。同一张幻灯片配什么图，AI 会根据这一页想传递的情绪来判断，而不是看到"数据"就搜"图表"。

## 怎么用

**前置条件：** 你需要有 Claude Code（或 OpenClaw、Cursor 等支持 skill/agent 的 AI 编码代理）。

**部署方式：** 不需要手动敲命令。把 GitHub 地址给你的 AI，告诉它：

> "部署 PPT 工作流，参考 GitHub README：https://github.com/wangjialiang678/ppt-workflow"

AI 会自己完成：克隆仓库、安装 skill、安装依赖（Node.js + Python）、提示你配置 API Key。

需要两个 API Key，都是免费的：
- **Gemini API Key**：用于 AI 配图，在 Google AI Studio 申请
- **Pexels API Key**：用于免费图库搜索，在 Pexels 官网申请

配置好之后，直接说：

> "做 PPT，主题是 XX，风格选 Apple Bento"

工作流就启动了。

## 七个阶段简介

工作流内部分七个阶段，AI 自动依次执行：

1. **大纲生成** — 根据主题和背景材料写 Markdown 大纲
2. **需求确认** — 和你对齐页数、受众、重点
3. **风格选择** — 从 15 种风格中匹配最合适的
4. **HTML 幻灯片生成** — 将大纲渲染成带样式的 HTML
5. **AI 配图** — 为每页选配或生成图片
6. **PPTX 转换** — 用 Playwright 截图转成真实 .pptx
7. **质量审核** — 自动检查对比度、布局、内容完整性
